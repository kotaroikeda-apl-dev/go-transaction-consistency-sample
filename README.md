# go-transaction-consistency-sample
This repository demonstrates how to design transaction boundaries and maintain strong consistency in financial domains using Go.

# 概要  
本リポジトリは、金融ドメインにおける整合性設計をテーマにしたサンプル実装です。  
特に以下の点にフォーカスしています。
- トランザクション境界はどこに置くべきか
- 集約はどのように整合性を守るのか
- エンティティはどこまで責務を持つべきか
- 「壊れない設計」とは何か

DDDのパターンをなぞることが目的ではなく、  
金融システムにおいて整合性をいかに守るか を主題としています。
